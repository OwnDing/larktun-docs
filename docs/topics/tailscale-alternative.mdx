# 云雀通是否可以作为 Tailscale 替代方案？

如果你的目标是把多台设备安全连成一张私有网络，`Tailscale` 是非常成熟的选择。云雀通 Larktun 继承了类似的组网思想，并针对国内用户和团队场景做了更贴近本地使用习惯的产品化设计。

所以，云雀通可以在很多场景下作为 Tailscale 替代方案，尤其适合希望中文界面、SaaS 控制台、多租户隔离和更清晰设备治理能力的用户。

## 适合使用云雀通的场景

- 远程访问家里的 NAS、Home Assistant、摄像头或 mini 主机。
- 在不暴露公网端口的情况下 SSH 到开发机或服务器。
- 团队通过 ACL 控制谁能访问哪台设备、哪个网段和哪个服务。
- 需要中文设备名、设备别名和更接近日常识别习惯的设备列表。
- 需要共享、专用或自建中继，以适配不同地域和网络质量。
- 希望使用不占用系统 VPN 权限的移动端 App 完成 Ping、文件传输、SSH 和 SFTP。

## 云雀通与 Tailscale 的差异

云雀通并不是要否定 `Tailscale`。相反，云雀通尊重它的技术思想和生态价值。

云雀通更关注这些方向：

- 本地化体验：中文设备名、中文控制台和更适合国内用户的说明。
- SaaS 多租户：一个用户或组织可以拥有独立租户，网络和策略彼此隔离。
- 权限治理：围绕 ACL、路由、中继和设备目录提供更清晰的管理界面。
- 移动端工具化：提供 App 内网络能力，在部分场景下不需要系统 VPN 权限。
- 国内网络适配：通过共享、专用或自建中继提升可达性和稳定性。

## 什么时候仍然适合直接使用 Tailscale

如果你已经完全适应 `Tailscale` 的账号体系、控制台和客户端生态，并且网络访问稳定、团队也熟悉它，那么继续使用 `Tailscale` 是合理的。

如果你希望更多掌控控制面，也可以研究 `Headscale` 自建方案。但自建意味着你需要维护服务器、域名、证书、中继、升级和故障排查。

云雀通适合希望减少自建成本，同时获得更本地化体验和权限治理能力的用户。
