# 开通第一条安全访问链路

建议用一个真实业务场景做首条链路验证，确保“租户隔离 + ACL + 中继 + 稳定性”四项都通过。

## 推荐示例

- 研发团队访问构建机或调试主机（SSH）
- 测试或支持团队访问远程桌面环境（RDP）
- 运维团队访问内部后台系统（Web）

## 配置流程

1. 选择目标设备，并确认它位于正确租户。
2. 为该租户创建或复用 ACL 规则，限制访问主体与时间窗。
3. 选择当前链路中继模式（共享、专用或自建）。
4. 发起访问并验证授权与连通性。
5. 检查会话日志、策略命中记录与稳定性指标。

## 验证清单

- 非授权用户是否被 ACL 拒绝
- 授权用户是否在跨网络条件下稳定访问
- 中继不可用时是否触发预期的重试或切换行为
- 客户端是否保持低占用且不干扰用户前台操作
- 审计记录是否完整可追溯
