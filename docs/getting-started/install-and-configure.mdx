# 部署方式与配置

Larktun 官网当前强调了“几分钟内搭建零信任网络”“共享中继 / 专用中继”“私有或混合部署”等能力。本页将这些信息整理成文档骨架，方便后续补齐正式交付说明。

## 推荐部署方式

### SaaS 云端控制台

适合快速试用、小团队接入和希望尽快完成首轮验证的场景。优点是上线速度快、基础设施负担小。

### 专用中继

适合成长型团队，需要比共享中继更可控的链路质量和容量边界，同时继续使用托管控制平面。

### 私有或混合部署

适合对合规、审计、内网控制边界和定制集成要求更高的团队。通常需要更明确的网络规划和交付流程。

## 环境规划

- 确认待接入设备分布，是办公终端、服务器还是跨地域研发节点
- 明确访问目标是远程桌面、SSH、VNC 还是内部系统入口
- 规划中继方式、出口区域、身份体系和日志留存方式
- 提前确认是否要接入企业 SSO、MFA、审批和告警推送

## 核心配置项

- 网络模式：Headscale 结合托管控制台，还是进一步采用私有/混合控制平面
- 中继策略：共享中继、专用中继或区域化链路规划
- 身份体系：账号密码、企业 SSO、MFA 与邀请机制
- 访问策略：按角色、项目、设备标签或时间窗口限制访问
- 审计配置：会话记录、安全事件、异常登录与越权行为监测

## 示意配置

以下配置仅用于说明部署思路，不代表最终的产品字段格式：

```yaml
deployment:
  controlPlane: saas
  relayMode: dedicated
  regions:
    - ap-east
    - ap-southeast
identity:
  sso: enterprise
  mfa: required
access:
  policyTemplate: corp-remote-access
  temporaryApproval: true
  expirationHours: 8
audit:
  sessionAnalytics: enabled
  securityCenter: enabled
  alertWebhook: https://example.com/hooks/security
```

## 配置建议

- 小团队试用时，先从托管控制台和默认接入能力开始，缩短首次上线时间
- 涉及审批、合规审计或跨客户资产管理时，优先规划身份体系与日志留存
- 对延迟、容量和网络边界更敏感的团队，尽早评估专用中继或私有/混合部署
- 在正式交付文档里，补齐每种模式的拓扑图、网络端口要求和故障回退方式
