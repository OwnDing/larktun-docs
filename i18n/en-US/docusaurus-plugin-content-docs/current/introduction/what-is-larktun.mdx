# Product Overview

This page is the right place to explain what Larktun is, what it helps teams accomplish, and how new users should think about it before they start integrating it.

## Problems it can solve

- Exposing local or internal services for secure external access
- Creating stable entry points for temporary or branch environments
- Supporting remote collaboration and troubleshooting without widening exposure too much

## Core value

- Manage service exposure, access control, and traffic records through a unified entry point
- Help teams establish verifiable paths between different environments quickly
- Create a shared language for future docs, blog posts, and showcase pages

## Common scenarios

- Remote access for QA and product teams to preview environments
- Minimal-surface exposure of internal systems by operations teams
- Temporary support access for diagnosing customer issues
