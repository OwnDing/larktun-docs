# 核心概念

为了保证文档、博客和案例描述一致，建议先统一以下几个核心术语。

## 项目或工作空间

用于隔离不同团队、业务线或环境的管理边界。后续的服务入口、访问策略和日志都可以围绕项目维度组织。

## 服务入口

服务入口用于对外提供统一访问地址，它通常负责承接域名、协议、鉴权方式和流量转发规则。

## 目标服务

目标服务是实际承载业务逻辑的应用实例，例如本地开发服务、测试环境 API 或企业内网后台。

## 访问策略

访问策略用于定义“谁可以访问、何时可以访问、从哪里访问”，建议在正式文档里进一步细化到角色、时效和审计规则。

## 诊断信息

诊断信息包括连接状态、访问日志、错误上下文和服务健康检查结果，是后续问题处理章节的重要基础。

## 建议的文档写法

- 先描述业务目标，再解释配置项含义
- 避免只写参数说明而缺少实际场景
- 在关键步骤后补充验证方法和失败后的排查入口
