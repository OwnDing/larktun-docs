# 核心概念

为了让介绍、上手和排障文档保持一致，建议先统一以下术语。

## 租户（Tenant）

租户是最核心的管理边界。每个租户都拥有独立网络、独立成员体系、独立 ACL 与独立审计数据。

## 隔离网络

租户网络默认互相隔离。即使在同一 SaaS 平台内，不同租户的设备和访问策略也不会自动互通。

## ACL（访问控制列表）

ACL 用来定义“谁在什么条件下可以访问哪些目标”。在 Larktun 中，ACL 按租户独立配置和生效。

## 中继服务器

当终端无法直连时，中继服务器承接流量转发。你可以根据需求选择共享中继、专用中继或用户自建中继。

## 客户端代理

安装在终端或服务器上的客户端负责建立加密链路并执行 ACL。其设计目标是低内存、低干扰、可长期稳定运行。

## 审计与诊断

审计数据包含会话记录、策略命中、异常事件与链路质量指标。诊断流程依赖这些数据定位问题根因。

## 文档写作建议

- 先说明租户边界，再说明 ACL 和网络路径
- 明确当前场景使用的是哪种中继模式
- 每个关键步骤都提供验证项，尤其是稳定性和资源占用观察点
