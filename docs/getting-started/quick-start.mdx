# 快速开始

本页帮助你在最短时间内完成一次有效验证：创建租户、配置独立 ACL、选择中继方式并验证稳定访问。

## 前置准备

- 一个可用的 Larktun 账号
- 至少一台待接入设备（电脑、服务器或跳板机）
- 一个要访问的目标服务（SSH、RDP、VNC 或内部 Web）
- 如需自建中继，提前准备一台公网可达主机

## 推荐路径

1. 创建首个租户（或工作空间），确认租户边界。
2. 接入首批设备，确认设备在线。
3. 配置该租户的 ACL 基线策略（最小权限优先）。
4. 选择中继模式：共享、专用或自建。
5. 发起首次访问验证并观察连通性。
6. 检查审计日志、策略命中和会话稳定性。
7. 在客户端观察内存占用，确认可长期后台运行。

## 示例配置（示意）

```yaml
tenant: engineering-prod
network:
  isolation: enabled
  relayMode: custom
  customRelay:
    endpoint: relay.example.com:443
acl:
  default: deny
  rules:
    - subject: group:devops
      action: allow
      resource: tag:build-node
      window: workhours
session:
  autoReconnect: true
agent:
  lowMemoryProfile: enabled
  runInBackground: true
audit:
  sessionLogs: enabled
  securityEvents: enabled
```
