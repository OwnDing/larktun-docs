# 诊断方法

建议使用固定流程定位问题，避免反复试错。

## 第一步：确认边界

- 问题是否只影响某个租户
- 是否只影响某类 ACL 规则或某一类设备
- 是否只发生在某个中继节点或某个网络区域

## 第二步：采集关键信号

- 租户 ID、策略版本与 ACL 命中记录
- 中继模式（共享/专用/自建）与节点健康状态
- 会话时延、重连次数、失败时间点
- 客户端内存占用与后台运行状态

## 第三步：分层定位

按“身份认证 -> ACL 决策 -> 中继路径 -> 目标服务”逐层排查，避免一次修改多个变量。

## 第四步：结论与固化

记录根因、短期绕过方案、长期修复方案，并更新相应租户模板与告警规则。

## 建议输出模板

- 影响范围：租户 / 用户组 / 设备标签
- 根因分类：策略、网络、中继、终端、目标服务
- 修复动作：立即动作与后续动作
- 回归检查：稳定性、低占用、无打扰体验是否恢复
