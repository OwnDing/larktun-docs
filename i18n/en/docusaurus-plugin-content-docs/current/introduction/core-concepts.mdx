# Core Concepts

To keep overview, onboarding, and troubleshooting content aligned, define the following terms first.

## Tenant

A tenant is the primary management boundary. Each tenant has its own network, members, ACL, and audit data.

## Isolated network

Tenant networks are isolated by default. Devices and policies from one tenant do not automatically connect to another tenant.

## ACL (Access Control List)

ACL defines who can access which targets under what conditions. In Larktun, ACL is scoped and enforced per tenant.

## Relay server

Relay servers forward traffic when direct paths are unavailable. Teams can use shared relay, dedicated relay, or user-managed relay.

## Client agent

The endpoint agent establishes encrypted paths and enforces ACL. It is designed for low memory usage, low interruption, and stable long-running sessions.

## Audit and diagnostics

Audit data includes session records, policy decisions, anomaly events, and path quality signals. Diagnostics depends on this data for root-cause analysis.

## Documentation guidance

- Describe tenant boundaries before policy and path details
- Explicitly state which relay mode a scenario uses
- Include validation checkpoints, especially around stability and resource usage
