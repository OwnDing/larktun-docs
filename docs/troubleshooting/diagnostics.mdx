# 诊断方法

建议把问题处理固定成一套可重复执行的流程，这样可以降低支持成本，也能让博客和案例页复用这些方法。

## 第一步：确认范围

- 问题只影响单个设备、单个工作空间，还是影响整组访问策略
- 问题只出现在某个地区、账号或时间窗口，还是普遍存在

## 第二步：采集信息

- 当前工作空间、策略模板与中继模式
- 设备在线状态、会话分析结果与延迟数据
- 请求时间、访问来源、错误日志与安全事件记录

## 第三步：定位链路

从身份验证、访问策略、中继链路到目标设备依次排查，避免同时修改多个变量导致问题难以复现。

## 第四步：形成结论

记录根因、临时缓解方案、长期修复方案，以及是否需要补充策略模板、监控规则或审计留存。
