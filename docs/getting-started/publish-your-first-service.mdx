# 发布第一个服务

当环境与基础配置完成后，建议用一个简单、可重复验证的服务完成首次发布。

## 推荐示例

- 一个本地开发中的 Web 应用
- 一个仅供测试人员访问的预发布接口
- 一个需要短期暴露给外部协作者的内部服务

## 发布流程

1. 选择目标服务并确认其本地可达。
2. 创建外部访问入口并绑定到目标服务。
3. 为入口配置访问策略，例如令牌、账号或来源限制。
4. 对外分享访问地址，并在有限范围内完成验证。
5. 记录访问日志、响应时间和异常信息。

## 验证清单

- 域名解析或入口地址是否正确
- 未授权访问是否被拒绝
- 授权用户是否可以稳定访问
- 日志中是否能看到完整请求与错误上下文
