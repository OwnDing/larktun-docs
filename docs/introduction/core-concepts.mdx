# 核心概念

为了让部署文档、使用手册和案例说明保持一致，建议先统一以下几个核心术语。

## 项目或工作空间

用于隔离不同团队、业务线或客户环境的管理边界。设备、访问策略、日志与成员授权都可以围绕工作空间组织。

## 设备

设备是加入 Larktun 网络的终端或节点，既可以是开发电脑、运维跳板机，也可以是服务器或桌面资产。

## Headscale 控制平面

Larktun 在官方介绍中以 Headscale 为底座扩展设备生命周期管理、细粒度访问控制和异常检测能力。你可以把它理解成网络控制与编排的基础层。

## 中继与链路

中继用于承接跨网络、跨地域访问流量。根据团队规模与安全要求，可以选择共享中继、专用中继，或进一步走向私有/混合部署。

## 远程访问策略

访问策略定义“谁可以访问哪类设备、在什么时间访问、是否需要审批、访问是否自动回收”。它是零信任治理的核心。

## 身份与安全能力

Larktun 支持多因素认证、企业 SSO、临时授权、审批和异常行为检测，帮助团队把身份安全融入日常访问流程。

## 诊断信息

诊断信息包括设备在线状态、会话分析、访问日志、错误上下文和安全事件，是问题处理章节的重要基础。

## 建议的文档写法

- 先描述业务场景，再解释它依赖的设备、策略和中继配置
- 对共享中继、专用中继、私有部署分别说明适用前提和限制
- 在每个关键步骤后补充验证方法、审计入口和回滚建议
