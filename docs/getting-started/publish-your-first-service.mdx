# 开通第一条安全访问链路

当工作空间与基础配置完成后，建议尽快用一个真实场景完成首条访问链路的验证，而不是停留在纯配置阶段。

## 推荐示例

- 研发团队通过 SSH 访问构建节点或调试主机
- 产品或测试团队通过 RDP 访问预发布桌面环境
- IT 运维团队为内网后台或桌面资产建立临时授权入口

## 配置流程

1. 选定目标设备或访问对象，并确认它已经成功接入工作空间。
2. 选择当前链路使用的中继方式与区域，确保访问路径清晰可控。
3. 为访问对象绑定策略模板，例如只允许指定用户组、固定时间窗或审批后访问。
4. 发起首次访问验证，测试 SSH、RDP、VNC 或内部系统入口是否可用。
5. 在控制台检查会话分析、延迟、日志与异常告警，确认该链路可继续复用。

## 验证清单

- 设备是否处于在线状态
- 未授权用户是否会被访问策略拒绝
- 授权用户能否在跨地域网络下稳定访问
- 会话日志、审计记录和安全事件中心是否能看到完整上下文
