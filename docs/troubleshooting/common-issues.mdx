# 常见问题

本页面向直接使用云雀通的用户，按“你看到的问题 -> 你可以怎么做”给出处理方法。

## 建议排查顺序

1. 先看 `Nodes`：源设备和目标设备是否在线。
2. 再看 `ACLs`：访问主体、目标设备、端口是否匹配。
3. 涉及子网时看 `Routers`：子网路由是否批准。
4. 有延迟或断连时看 `Derper`：是否需要切换专用或自定义中继。

## 1. 访问被拒绝（ACL）

你可能会看到：

- 提示无权限访问
- 设备在线但连接失败
- 只有个别用户可访问，其他用户被拒绝

你可以这样做：

1. 在 `ACLs` 确认“访问主体”是否是你当前账号或所属用户组。
2. 确认“访问目标”是否是正确设备，而不是同名或旧设备。
3. 确认协议与端口是否正确（例如 `SSH 22`、`HTTPS 443`、`RDP 3389`）。
4. 若刚修改规则，等待几秒后重试一次连接。

## 2. 设备显示离线或无法连接

你可能会看到：

- `Nodes` 中目标设备离线
- 设备刚在线又掉线
- 一直连接超时

你可以这样做：

1. 在目标设备本机确认客户端已登录且未手动断开。
2. 在 `Nodes` 刷新状态，确认设备名称和你要访问的是同一台。
3. 切换网络后重试（例如 Wi-Fi 与有线网络互测）。
4. 若长期离线，重启目标设备客户端后再测试。

## 3. 子网资源访问失败（仅子网场景）

你可能会看到：

- 能访问节点本机，但不能访问其后面的子网设备
- 子网 IP 无法连通

你可以这样做：

1. 在 `Routers` 确认对应子网路由已批准。
2. 确认目标子网段填写正确，没有地址冲突。
3. 重新从源设备访问一次目标子网资源。

## 4. 延迟高或偶发断连

你可能会看到：

- 远程桌面卡顿
- SSH 偶发中断
- 访问时好时坏

你可以这样做：

1. 先确认是否在高峰网络时段出现问题。
2. 免费中继默认对所有用户可用，无需手动选择。
3. 如果对延迟和稳定性要求更高，在 `Derper` 使用专用中继。
4. 如有网络边界要求，在 `Derper` 配置自定义中继。

## 5. 客户端占用偏高或“打扰感”明显

你可能会看到：

- CPU 或内存占用偏高
- 频繁弹窗或打断当前操作

你可以这样做：

1. 保持客户端在后台运行，不要反复手动重连。
2. 关闭不必要的前台窗口，避免重复打开多个会话。
3. 在“设置”中开启更安静的运行方式（如最小化到托盘）。

## 6. 自定义中继不可用（仅自定义中继场景）

你可能会看到：

- 自定义中继节点不可达
- 配置后连接反而更差

你可以这样做：

1. 在 `Derper` 核对 `hostName`、`ipv4`、`derpport`、`stunport` 是否填写正确。
2. 临时切回默认免费中继，确认链路是否恢复。
3. 恢复后再逐项检查自定义中继配置与网络放行策略。

## 什么时候需要联系支持

如果按以上步骤仍无法解决，请提交以下信息以便快速定位：

- 问题发生时间
- 你的账号与租户名称
- 源设备与目标设备名称
- 使用协议与端口
- 你已尝试过的步骤
