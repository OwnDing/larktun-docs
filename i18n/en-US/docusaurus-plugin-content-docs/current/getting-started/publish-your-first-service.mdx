# Publish Your First Service

Once the environment and baseline configuration are ready, publish a simple service that is easy to verify and repeat.

## Suggested examples

- A locally running web application
- A preview API only shared with QA
- An internal service that needs short-term external access

## Publishing flow

1. Select the target service and confirm it is reachable locally.
2. Create an external entry point and bind it to the service.
3. Add an access strategy, such as tokens, accounts, or source restrictions.
4. Share the address with a limited audience and complete verification.
5. Record logs, response times, and any abnormal signals.

## Verification checklist

- Domain resolution or entry point address is correct
- Unauthorized access is blocked
- Authorized users can access the service reliably
- Logs contain enough request and error context
