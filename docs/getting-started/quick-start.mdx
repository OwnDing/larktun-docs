# 快速开始

本页用于承载最短接入路径，帮助新用户从注册账号到接入首批设备，再到完成第一次安全访问验证。

## 前置准备

- 一个 Larktun 账号
- 至少一台待接入的设备，例如开发电脑、跳板机或测试服务器
- 一条需要验证的访问链路，例如 SSH、RDP、VNC 或内部 Web 系统
- 如果你已有 Headscale 网络，提前准备现有网络和身份策略信息

## 推荐路径

1. 注册账号并创建首个工作空间。
2. 在免费额度内接入首批设备，确认设备在线状态正常。
3. 选择共享中继或当前可用的默认接入方式，建立第一条访问链路。
4. 为链路配置身份与访问策略，例如临时授权、审批或基础 MFA。
5. 使用 SSH、RDP、VNC 或内部系统访问完成验证。
6. 打开会话分析、日志与审计记录，确认链路稳定且可追溯。

## 占位配置示例

下面的配置作为部署思路示意，字段命名请以后续控制台或 CLI 的真实版本为准。

```yaml
workspace: engineering-demo
network:
  mode: headscale
  relay: shared
identity:
  mfa: enabled
  sso: optional
policy:
  template: remote-dev
  temporaryApproval: true
audit:
  sessionLogs: enabled
  securityEvents: enabled
```
