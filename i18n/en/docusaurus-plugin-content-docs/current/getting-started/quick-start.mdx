# Quick Start

This page helps you complete one effective first validation: create a tenant, apply tenant ACL, choose relay mode, and verify stable access.

## Prerequisites

- An active Larktun account
- At least one device to onboard (laptop, server, or bastion)
- A target service to reach (SSH, RDP, VNC, or internal web)
- If using user-managed relay, one publicly reachable host

## Recommended flow

1. Create your first tenant (or workspace) and confirm the boundary.
2. Onboard initial devices and confirm online state.
3. Configure tenant ACL baseline (least privilege first).
4. Select relay mode: shared, dedicated, or user-managed.
5. Run the first access validation and confirm connectivity.
6. Verify audit logs, ACL decisions, and session stability.
7. Observe client memory profile for long-running background use.

## Example configuration (illustrative)

```yaml
tenant: engineering-prod
network:
  isolation: enabled
  relayMode: custom
  customRelay:
    endpoint: relay.example.com:443
acl:
  default: deny
  rules:
    - subject: group:devops
      action: allow
      resource: tag:build-node
      window: workhours
session:
  autoReconnect: true
agent:
  lowMemoryProfile: enabled
  runInBackground: true
audit:
  sessionLogs: enabled
  securityEvents: enabled
```
