# FAQ

## When does a question belong in the FAQ?

If the same question comes up repeatedly and can usually be answered in a few sentences, it is a good candidate for the FAQ.

## When should it become a standalone page?

If the answer needs detailed steps, sample configurations, risk notes, or multiple scenario branches, it should become its own document.

## How should the FAQ be organized?

Grouping by phases such as "before setup", "during setup", "after launch", and "during troubleshooting" keeps navigation simple.
