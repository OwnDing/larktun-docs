# Diagnostics

Use a fixed diagnostic flow to avoid repeated trial-and-error.

## Step 1: confirm scope

- Is the issue limited to one tenant?
- Is it tied to a specific ACL rule set or device category?
- Is it isolated to a relay node or region?

## Step 2: collect key signals

- Tenant ID, policy version, and ACL decision records
- Relay mode (shared/dedicated/user-managed) and node health
- Session latency, reconnect count, and failure timestamps
- Client memory usage and background runtime state

## Step 3: troubleshoot by layer

Inspect in order: identity validation -> ACL evaluation -> relay path -> destination service. Avoid changing multiple variables at once.

## Step 4: document and harden

Record root cause, short-term mitigation, long-term fix, and update tenant templates plus alerting rules.

## Suggested report template

- Impact scope: tenant / user group / device tags
- Root-cause class: policy, network, relay, endpoint, destination
- Remediation: immediate actions and follow-up actions
- Regression checks: stability, low footprint, non-intrusive behavior
