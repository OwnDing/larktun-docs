# Common Issues

This page focuses on early-stage blockers: ACL denials, unavailable user-managed relay, abnormal client resource usage, and unstable sessions.

## Access denied by ACL

First verify the request is made in the correct tenant, then inspect ACL match status, grant expiration, and approval completion.

## User-managed relay unavailable

Validate relay endpoint and port, node health status, and network constraints such as firewall or certificate issues.

## Higher-than-expected client usage

Confirm low-memory profile is enabled and check for abnormal retry loops or excessive logging levels.

## Intermittent disconnects

Check network quality and relay path first, then verify reconnect behavior and destination host load.

## End-user reports disruption

Confirm client mode is silent background runtime and review whether any unnecessary prompts or foreground interruptions are configured.
